# Лабораторная работа №3
* **
## Easy + Флаги/компараторы. Структура данных 1.a
* **
## Цель:
* Написать алгоритмический мини-пакет (структуры данных и сортировки) 
* **
## Допущения:
* Для bucket sort флаги обязаны возвращать числовое значение (int или float)
* Аналогично поэтому для bucket sort нельзя использовать многоуровневые сортировки
* **
## Алгоритм решения
* Пользователь вводит команду с аргументами (Ввод в бесконечном цикле)
* Команда обрабатывается. Проверяется, переключается ли режим
* Если режим sort, то при помощи eval() обрабатывается строка (Учитываются банворды). Вызывается соответствующая функция сортировки
* Если режим node, то создается LinkedList(). Далее по командам вызываются методы связного списка
* **
## Документация
* node_mode - переход в node_mode (Можно работать со связным списком)
* sort_mode - переход в sort_mode (Работают сортировки)
* ## Режим sort:
* bubble_sort(a: list, comp, flag)
* quick_sort(a: list, comp, flag)
* radix_sort(a: list[int], base)
* counting_sort(a: list[int])
* bucket_sort(a: list, comp, flag)
* heap_sort(a: list, comp, flag)
* comp - компаратор (Задавать через лямбда-функции). 
* flag - флаг (задавать через лямбда-функции)
* Для флага и компаратора есть значение по-умолчанию
* a - список для сортировки
* ## Режим node:
* {} - переменные
* pop - удалить последний элемент и вывести значение
* peek - посмотреть последнее значение
* is_empty - проверить, является ли связный список пустым
* push {value} - вставить в конец списка value
* len - вывести длину списка
* min - вывести минимальный элемент списка
